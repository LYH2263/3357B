
# 二、0-1 代码生成题（15 道）

### Z1. 在线测验考试模块
为教学平台从零构建一套在线测验系统。教师在后台创建试卷：设置试卷标题、关联的课程介绍、限时时长、及格线，并录入若干道单选/多选题（题干、选项、正确答案、分值），试卷有「草稿/已发布/已截止」状态；学生在学生中心看到已发布且在有效期内的试卷，进入答题页后限时作答（展示倒计时，超时自动交卷），提交后系统按标准答案自动判分，记录得分、用时与每题对错，并向学生展示成绩与错题解析；同一份试卷可限制每位学生的作答次数（如仅一次或取最高分）。教师可查看某试卷的全部学生成绩列表、平均分、及格率与每题正确率分析。请设计相关数据表（试卷、题目、选项、作答记录）、后端接口（试卷增删改与发布、取题作答、自动判分、成绩统计）与前后端页面。需要处理多选题的判分规则（全对才得分或按比例）、超时与重复提交、已截止试卷不可再答、以及试卷发布后题目变更的处理策略。

### Z2. 公告通知中心模块
从零构建一套面向全校的公告通知中心。教师在后台发布公告：填写标题、富文本/纯文本正文、目标范围（全体或指定一个/多个班级）、是否置顶、生效与失效时间，公告支持草稿与发布两种状态、可编辑与撤回；学生端导航栏出现「铃铛」入口并显示个人未读公告数，点击展开公告列表（置顶优先、其余按发布时间倒序），阅读某条后标记为已读且未读数实时递减，失效公告自动从列表隐去。系统需为每个学生维护其对每条公告的已读状态，并保证「目标范围」决定了哪些学生可见某条公告。请设计数据表（公告、公告-已读记录、公告-目标班级关联）、后端接口（公告增删改发布撤回、按学生计算可见与未读、标记已读、未读计数）与教师端管理页、学生端通知中心页。需要处理目标范围与班级变更时可见集合的重算、置顶数量与生效时间窗口、以及大量学生下未读统计的效率问题。

### Z3. 作业提交与批改模块
从零构建作业全流程模块。教师布置作业：标题、要求说明、所属班级、截止时间、满分分值；学生在学生中心看到分配给本班的作业及其状态（未提交/已提交/已批改/逾期），在截止时间前上传作业文件（复用文件上传）并可在截止前覆盖重交，截止后不可再交；教师在后台查看某作业的提交情况列表（已交/未交一目了然），逐份下载文件、打分并填写评语，支持打回要求重交（在截止前）；学生可查看自己作业的分数与评语。请设计数据表（作业、作业提交记录）、后端接口（布置作业、学生提交/重交、教师批改、查询提交清单与统计）与前后端页面。需要处理截止时间的服务端权威判定（不信任前端时间）、重复提交时旧文件的处理、逾期未交的自动标记、以及未提交学生名单的计算。该模块聚焦文件作业的提交批改，不含在线答题判分。

### Z4. 教学内容评价评分模块
从零构建针对教学内容的评价系统。学生在学生端「教学内容」每一条目下可进行评分（1–5 星）并撰写文字评价，约束为每位学生对同一条教学内容仅能评价一次但可修改或删除自己的评价；某条教学内容的详情处展示其平均星级、评价总数与评价列表（含评价人、星级、内容、时间，按时间倒序并支持分页）；教师后台可查看各教学内容的平均分与评价数排行，能够下架（删除）违规评价并查看被删记录。请设计数据表（评价记录，含唯一性约束保证一人一评）、后端接口（提交/修改/删除评价、计算平均分与分布、分页拉取评价、教师端排行与管理）与前后端页面。需要处理平均分的实时计算与展示精度、同一用户重复提交的幂等更新、空评价（仅打星不写文字）是否允许、以及教师删除评价后统计同步等问题。

### Z5. 选课模块
从零构建学生选课系统。教师在后台发布可选课程：课程名称、简介、授课教师、容量上限、选课起止时间，课程具备「开放选课/已截止」状态；学生在学生中心浏览可选课程列表（展示剩余名额），在选课时间内进行选课，当某课程剩余名额为 0 时显示「已满」且不可再选，学生可在截止前退选释放名额；学生查看「我的已选」课程，教师查看每门课程的选课名单、已选人数与剩余名额，并可导出选课名单。核心难点在于并发选课不得超过容量上限。请设计数据表（可选课程、选课记录，含防重复选课约束）、后端接口（发布课程、选课/退选、查询可选与已选、名单与名额统计）与前后端页面。需要处理高并发下名额的原子扣减（避免超额）、重复选课的幂等、退选后名额回补、以及选课时间窗口的服务端校验。

### Z6. 考勤签到模块
从零构建课堂考勤签到系统。教师在后台对某个班级发起一次签到任务：设定签到主题与有效时长（如 5 分钟），任务在有效期内为「进行中」，超时自动「结束」；该班学生在学生中心于有效期内点击「签到」完成出勤记录，逾期则无法签到；签到结束后教师查看该次签到的出勤名单、出勤人数与出勤率，未签到的学生自动计为「缺勤」，教师可对个别学生手工补登（如「请假」）。教师还可按班级查看历史签到记录与各学生的累计出勤率。请设计数据表（签到任务、签到/出勤记录）、后端接口（发起签到、学生签到、结束与统计、历史查询）与前后端页面。需要处理有效期的服务端权威判定、重复签到的幂等、班级成员与缺勤名单的计算、并发签到写入，以及任务结束后状态与统计的一致性。

### Z7. 站内私信模块
从零构建学生与教师之间的一对一私信系统。学生可选择一位教师发起私信会话并发送消息，教师在后台看到自己的会话列表（按最近活跃排序，展示对方姓名、最后一条消息预览、未读数），进入某会话后双方消息以聊天气泡按时间排列、区分发送方，进入会话即把对方发来的未读消息标记为已读；双方均可在会话中持续往返发送文本消息。会话以「学生-教师」二元组聚合，避免重复创建。请设计数据表（会话、消息）、后端接口（发起或获取会话、发送消息、拉取会话列表与未读数、拉取会话内消息并标记已读）与教师端、学生端的会话界面。需要处理会话的去重与查找、未读计数的准确维护、消息分页加载、以及一方不存在或被删除时的健壮处理。该模块为点对点私信，不涉及公共讨论或群发公告。

### Z8. 学习进度跟踪模块
从零构建学习进度跟踪系统。学生在学生端「教学内容」列表中可将每条标记为「已学完」或取消标记，系统为每位学生维护其对各教学内容的完成状态；学生端首页展示个人总体学习进度条（已学完数 / 教学内容总数的百分比）与「已学完」清单，并可按完成时间回顾学习轨迹；教师后台可查看每位学生的学习进度、每个班级的平均完成率，以及每条教学内容被多少学生学完的「热度」排行。请设计数据表（学习进度记录）、后端接口（标记/取消完成、计算个人与班级进度、内容维度热度统计）与前后端页面。需要处理总数变化（新增教学内容）时进度百分比的动态重算、标记操作的幂等、未学完与已学完清单的区分、以及班级平均进度的聚合效率。该模块聚焦学习状态跟踪，不涉及打分或测验。

### Z9. 资源收藏夹模块
从零构建跨资源类型的收藏夹系统。学生在学生端浏览教学内容与实验内容时，每条可一键收藏或取消收藏，按钮即时反映当前收藏状态；学生端新增「我的收藏」页，统一展示其收藏的教学内容与实验内容（按资源类型分区展示，标注收藏时间，可从此处直接下载附件或取消收藏）；系统需区分被收藏资源的类型与主键，保证同一资源不被重复收藏。教师后台可查看「最受欢迎资源」榜单（按被收藏次数对教学内容与实验分别排行）。请设计数据表（收藏记录，含资源类型 + 资源 ID 的唯一约束）、后端接口（收藏/取消、查询某学生的收藏清单、按类型聚合的收藏排行）与前后端页面。需要处理多态资源（两种不同表）的统一收藏存储、被收藏资源已被删除时的清理或灰显、收藏状态在列表与收藏页间的一致性，以及重复收藏的幂等。

### Z10. 积分与班级排行榜模块
从零构建学生积分与排行榜系统。系统依据学生的若干行为发放积分（例如提交提问、完成签到、按时提交作业等，积分规则集中在一处可调整配置），累计每位学生的总积分并保留积分变动明细（来源、分值、时间）；学生端展示个人当前积分、积分明细流水，以及「班级排行榜」（同班学生按积分降序，高亮自己的名次与积分）；教师后台可查看全校积分总榜与各班榜单，并可对个别学生手工加分或扣分（需填写理由，计入明细）。请设计数据表（学生积分汇总、积分明细）、后端接口（积分发放与手工增减、查询个人积分与明细、班级与全校排行）与前后端页面。需要处理积分发放的幂等（同一行为不重复计分）、规则集中化与可维护性、并列名次的处理、积分汇总与明细的一致性，以及排行榜的查询效率。该模块聚焦积分计量与排名，不直接耦合任何单一业务模块的内部实现。

### Z11. 班级讨论区论坛模块
从零构建班级公共讨论区。同班的学生与任课教师可在本班讨论区发帖（标题 + 正文），其他成员可对帖子回帖，帖子与回帖均可点赞（每人对同一帖子/同一回帖只能点赞一次，可取消）；帖子列表展示作者、发布时间、回帖数、点赞数，并按「最新回复」排序，支持置顶（教师可置顶本班帖子）；帖子详情页展示正文与全部回帖（按时间正序，分页加载），发帖人或教师可删除自己的帖子与回帖（删除帖子需级联处理其回帖与点赞）。请设计数据表（帖子、回帖、点赞，点赞含唯一约束）、后端接口（发帖/回帖/删除、点赞与取消、列表与详情分页、置顶）与前后端页面。需要处理点赞的幂等与计数准确、按最新回复排序的实现、删除时的级联清理、空讨论区与权限边界（只能操作本班、只能删自己的内容，教师有更高权限）。该模块为公开多人讨论，区别于点对点私信。

### Z12. 教师课表与日程模块
从零构建教师课表与日程系统。教师在后台维护个人周课表，以「周一至周日 × 若干节次」的网格呈现，每个单元格可填写课程名称、授课班级、上课地点，支持新增、编辑、删除单节安排；同一教师在同一时间段不允许重复排课（冲突时给出提示）；提供「按班级查看课表」的视图（汇总各教师在该班的安排生成班级课表），学生端可查看本班课表；课表支持按周次切换查看，并能复制上一周课表到本周以便快速排课。请设计数据表（课程安排，含教师、星期、节次、班级、地点）、后端接口（增删改单节、冲突校验、按教师/按班级查询、复制周课表）与前后端网格页面。需要处理时间段冲突的服务端校验、班级课表由多教师安排汇总时的同格冲突展示、空课表占位、以及复制课表时的覆盖策略。该模块聚焦排课与课表展示，不涉及考勤或选课。

### Z13. 学生请假申请审批模块
从零构建学生请假申请与审批流程。学生在学生中心提交请假申请：请假类型（事假/病假/其他）、起止日期、事由说明，可附上传证明文件，提交后状态为「待审批」，在被处理前可撤回；教师在后台看到本班待审批申请列表（按提交时间排序），可「批准」或「驳回」并填写审批意见，状态相应流转为「已批准/已驳回」；学生在学生端查看自己的请假记录及其最新状态与教师意见，教师可按状态与时间筛选并查看本班请假统计（如各类型请假次数）。请设计数据表（请假申请，含状态机字段）、后端接口（提交/撤回、审批、按学生与按班级查询、统计）与前后端页面。需要处理状态流转的合法性校验（只有「待审批」可被审批或撤回）、起止日期的合理性校验、证明文件的可选上传、并发审批与重复操作的幂等，以及只能审批本班、只能撤回本人申请的权限边界。

### Z14. 学生综合档案与电子成绩单模块
从零构建学生综合档案模块。系统为每位学生聚合其综合信息形成一页式电子档案：基本资料（姓名、学号、性别、班级、描述、头像）、状态信息（审核状态、是否优秀）、互动概况（累计提问数、最近一次提问）等，数据来自现有的学生、班级、互动等表的聚合；教师在「学生管理」中可对某学生「查看档案」打开档案页，信息分区清晰展示；档案支持导出为适合打印的版式（A4 排版，含学校抬头、生成时间与页脚），供下载或直接打印。教师还可对一个班级批量生成档案册（合并多名学生档案为可打印文档）。请设计后端聚合查询接口（按学生 ID 汇总多表信息、按班级批量汇总）与前端档案页/打印版式页面，必要时引入轻量的服务端聚合或前端打印样式。需要处理多表聚合的性能、缺失字段（无头像/无描述/无提问）的占位、打印分页与样式、以及批量导出时的数据量控制。该模块聚焦信息聚合与档案呈现，不涉及成绩录入。

### Z15. 全站统一搜索模块
从零构建一个跨模块的全站搜索。学生端导航栏提供全局搜索框，输入关键字后进入统一结果页，在一个页面内分类展示命中的教学内容、实验内容、课程介绍、技术动态、教师信息等多类结果，每类展示命中数量、可展开查看若干条并点进对应详情，无任何命中时给出空状态与搜索建议。后端需提供一个聚合搜索接口，单次请求即返回各类目的命中结果与计数（在数据库层完成模糊匹配），避免前端对各列表分别拉全量再过滤；结果页支持按类目切换 Tab、对每个类目内部分页加载，并对命中关键字做高亮。请设计后端聚合搜索接口（统一入参关键字，分类返回结构化结果与计数）与前端搜索框、结果页。需要处理多表查询的性能与并行、空关键字与超长关键字的边界、关键字高亮时的转义、各类目结果的分页协调，以及未来扩展新类目时接口的可维护性。该模块为只读检索聚合，不修改任何业务数据。
